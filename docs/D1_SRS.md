# Software Requirements Specification (SRS)  
## ระบบตัวกลางสำหรับหาและปล่อยห้องเช่า

---

## 1. บทนำ (Introduction)

### 1.1 Purpose
เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายความต้องการของระบบตัวกลางสำหรับการค้นหาและปล่อยห้องเช่า  
โดยมีวัตถุประสงค์เพื่อใช้เป็นแนวทางในการพัฒนา ทดสอบ และประเมินระบบ

### 1.2 Scope
ระบบตัวกลางสำหรับหาและปล่อยห้องเช่าเป็น **Web Application** ที่ช่วยเชื่อมต่อระหว่าง
- ผู้ที่ต้องการปล่อยเช่าห้อง  
- ผู้ที่ต้องการเช่าห้อง  

โดยผู้ใช้งานสามารถค้นหาห้องเช่า ดูรายละเอียด ติดต่อผู้ปล่อยเช่า  
และจัดการประกาศได้ผ่านระบบ

### 1.3 Definitions, Acronyms, Abbreviations
- **SRS** – Software Requirements Specification  
- **User** – ผู้ใช้งานระบบ  
- **Renter** – ผู้เช่าห้อง  
- **Owner** – ผู้ปล่อยเช่าห้อง  
- **Admin** – ผู้ดูแลระบบ  

---

## 2. Overall Description

### 2.1 Product Perspective
- เป็น Web Application  
- ทำงานผ่าน Web Browser  
- มีระบบฐานข้อมูลสำหรับจัดเก็บข้อมูลผู้ใช้และห้องเช่า  
- ระบบทำหน้าที่เป็น “ตัวกลาง” ไม่เกี่ยวข้องกับการชำระเงินจริง  

### 2.2 User Classes and Characteristics

| User Type | Description |
|---------|------------|
| Guest (ผู้เยี่ยมชม) | ดูรายการห้องเช่าได้ |
| Renter (ผู้เช่า) | ค้นหา ดูรายละเอียด และติดต่อผู้ปล่อยเช่า |
| Owner (ผู้ประกาศขาย) | ลงประกาศ แก้ไข และลบห้องเช่า |
| Admin (ผู้ดูแล) | จัดการระบบและตรวจสอบประกาศ |

### 2.3 Operating Environment
- **OS:** Windows / macOS / Linux  
- **Browser:** Chrome, Firefox  
- **Server:** Web Server + Database Server  

### 2.4 Design and Implementation Constraints
- ต้องใช้งานผ่าน Web Browser  
- ผู้ใช้ต้องสมัครสมาชิกก่อนติดต่อหรือปล่อยเช่า  
- ข้อมูลต้องถูกจัดเก็บในฐานข้อมูล  

---

## 3. Functional Requirements

### 3.1 User Management
- **FR-01** ระบบต้องให้ผู้ใช้สมัครสมาชิกได้  
- **FR-02** ระบบต้องให้ผู้ใช้เข้าสู่ระบบและออกจากระบบได้  
- **FR-03** ระบบต้องแยกสิทธิ์ผู้ใช้งานระหว่าง Renter, Owner และ Admin  

### 3.2 Room Listing Management (ผู้ปล่อยเช่า)
- **FR-04** ผู้ปล่อยเช่าต้องสามารถเพิ่มข้อมูลห้องเช่าได้  
- **FR-05** ผู้ปล่อยเช่าต้องแก้ไขข้อมูลห้องเช่าได้  
- **FR-06** ผู้ปล่อยเช่าต้องลบประกาศห้องเช่าได้  
- **FR-07** ระบบต้องแสดงสถานะห้องเช่า (ว่าง / ไม่ว่าง)  

### 3.3 Room Searching (ผู้เช่า)
- **FR-08** ผู้ใช้ต้องค้นหาห้องเช่าตามเงื่อนไขได้ (ราคา, ทำเล, ประเภทห้อง)  
- **FR-09** ระบบต้องแสดงรายละเอียดห้องเช่า  
- **FR-10** ผู้เช่าต้องสามารถติดต่อผู้ปล่อยเช่าผ่านระบบได้  

### 3.4 Admin Management
- **FR-11** ผู้ดูแลระบบต้องจัดการบัญชีผู้ใช้ได้  
- **FR-12** ผู้ดูแลระบบต้องตรวจสอบและลบประกาศที่ไม่เหมาะสมได้  

---

## 4. Non-Functional Requirements

### 4.1 Performance
- ระบบต้องรองรับผู้ใช้งานพร้อมกันอย่างน้อย 100 คน  
- หน้าเว็บต้องโหลดไม่เกิน 3 วินาที  

### 4.2 Security
- รหัสผ่านผู้ใช้ต้องถูกเข้ารหัส  
- ผู้ใช้ต้องล็อกอินก่อนใช้งานฟังก์ชันสำคัญ  

### 4.3 Usability
- ระบบใช้งานง่าย ไม่ซับซ้อน  
- รองรับการใช้งานบนมือถือและคอมพิวเตอร์  

### 4.4 Reliability
- ระบบต้องไม่สูญหายของข้อมูล  
- ข้อมูลต้องถูกต้องและอัปเดตล่าสุด

## 7. Assumptions and Dependencies
- ผู้ใช้งานมีอินเทอร์เน็ต  
- ระบบไม่รองรับการชำระเงินโดยตรง  
- ข้อมูลขึ้นอยู่กับความถูกต้องจากผู้ปล่อยเช่า
